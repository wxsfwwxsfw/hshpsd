# 怪物资源替换 SOP

这份说明用于以后替换怪物美术资源时先阅读，避免重复踩这次猪资源的坑。

## 目标

- 先保证源图方向正确
- 再保证进游戏后清晰、不拉伸、不串帧
- 最后才调动作节奏和表现

## 先看源图，不要急着进项目

建议先把候选图放到外部目录里看，例如 `d:\new\pig`。

进入项目前先确认这几件事：

- 风格是否和现有敌人一致，优先看轮廓、描边、细节密度，不要只看大图观感
- 是否适合小尺寸显示，不要拿大插画直接当精灵图
- 是否已经是透明背景
- 帧与帧之间是否留了足够安全边距

## 分清两种尺寸

一定要分清：

- 资源尺寸：`png/plist` 里每一帧的真实尺寸
- 显示尺寸：脚本里节点最终显示到场景里的尺寸

不要把这两件事混在一起调。

正确顺序是：

1. 先把资源做清晰
2. 再在脚本里调显示大小

## 推荐流程

1. 先在外部目录确认候选源图
2. 扣底，确保背景透明
3. 裁掉上下左右大空白
4. 单独导出每一帧检查是否串帧
5. 确认单帧没问题后再组回 atlas
6. 安装到 `assets/resources/<monster>/...`
7. 进入游戏后只调显示比例、位置、动画速度

## 这次猪资源踩过的坑

### 1. 大图硬缩进小精灵会糊

问题：

- 大图插画缩到几十像素会发虚
- 游戏里再显示时会更明显

做法：

- 优先使用接近游戏实际尺寸感的精灵表
- 先确认缩小后的单帧还清楚，再安装到游戏

### 2. 资源尺寸和显示尺寸混用

问题：

- 直接把资源从 `55x55` 改成 `40x40` 会进一步损失质量

做法：

- 保留较清晰的源帧
- 只在脚本里调显示比例

### 3. 非正方形怪物不能强行按正方形显示

问题：

- 猪单帧接近 `72x56`
- 如果显示层写成同宽同高，会横向拉伸

做法：

- 显示宽高按原始宽高比算
- 这类角色不要默认 `width === height`

### 4. 固定平均宽度切帧容易串图

问题：

- 相邻帧靠太近时，牙和尾巴会互相切进去

做法：

- 先导出单帧检查
- 必要时人工定切线，或者直接手工修单帧
- 最稳的是让源图本身留足透明间隔

### 5. 半像素位置会发虚

问题：

- 资源和显示比例没问题时，仍可能因为坐标落在半像素而模糊

做法：

- 角色位置做整数像素对齐

### 6. 先解决清晰度，再解决动作感

推荐顺序：

1. 单帧清晰
2. 比例正常
3. 显示尺寸合适
4. 再调动画帧数和速度
5. 最后补移动、攻击、受击等动作

## 行为表现复用原则

如果是给怪物补提示、表情、预警，不要先重写状态机。

先检查现有逻辑里有没有可复用的阶段和组件，例如：

- 现有攻击前摇状态
- `Expression` 表情系统
- 现有 FSM 切换点

优先在现有逻辑上加可视化，而不是重做行为。

## 本项目里换怪物前要先检查

- `assets/resources/<monster>/idle`
- `assets/resources/<monster>/move`
- `assets/resources/<monster>/attack`
- `assets/scripts/entity/<monster>/<Monster>.ts`
- `assets/scripts/entity/<monster>/<Monster>Fsm*.ts`

如果要替换猪资源，先看：

- `assets/scripts/entity/pig/Pig.ts`
- `assets/scripts/entity/pig/PigFsmForIdle.ts`

## 铺在整块地砖上的资源

像水面、陷阱、污渍这类“铺在地砖上的平面资源”，不要默认整格 `100%` 铺满。

原因：

- 地砖本身通常有视觉缝隙和内边距
- 资源如果按整格硬铺，会看起来像超出格子
- atlas 边缘采样也会放大这个问题

推荐顺序：

1. 资源源图分辨率先接近当前格子尺寸
2. 图集帧边缘做 padding / extrude
3. 显示层不要默认 `width === tileWidth`、`height === tileWidth`
4. 先把整体缩进一点，再做局部坐标微调

当前水块基准：

- `assets/scripts/entity/water/Water.ts`
- 水块显示尺寸按 tile 宽高收进约 `12%`
- 以后同类地砖覆盖资源先按这个显示比例试

L8 水块对齐基准：

- `x1y2`: `offsetX 14`, `offsetY -6`
- `x2y2`: `offsetX 8`, `offsetY -6`
- `x3y2`: `offsetX 2`, `offsetY -6`
- `x4y2`: `offsetX -4`, `offsetY -6`
- `x2y3`: `offsetX 8`
- `x3y3`: `offsetX 2`
- `x4y3`: `offsetX -4`

以后如果再做“整块地砖上的资源”位置调整，先以这组观感为准，不要重新从 0 开始试。

## 这次猪资源的最终经验

- 先看外部源图，再进项目
- 先验单帧，再装图集
- 资源保持清晰，显示尺寸在脚本里调
- 保持角色原始宽高比
- 位置做像素对齐
- 串帧问题优先检查切线和源图边距
- 警告类表现优先复用现有系统
